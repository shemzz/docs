## **Overview**

Logs provide detailed information about every conversation your agent has. Use logs to debug issues, understand agent behavior, and improve performance.

## **Log Information**

### **Conversation Details**

- **Call Information**: Date, time, duration
- **Participant Info**: Phone numbers, names
- **Conversation Flow**: Full transcript
- **Execution Path**: Which nodes were executed

### **Agent Behavior**

- **Responses**: What agent said
- **Decisions**: Choices made
- **Actions**: Actions executed
- **Variables**: Variables used

### **Performance Metrics**

- **Response Times**: How fast agent responded
- **Processing Time**: Time to process requests
- **Action Duration**: Time for actions to complete
- **Total Duration**: Full conversation time

## **Filtering and Search**

### **Filters**

Filter logs by:
- **Date Range**: Specific time period
- **Agent**: Specific agent
- **Call Type**: Inbound/Outbound
- **Status**: Success/Failed
- **Duration**: Call length

### **Search**

Search logs for:
- **Keywords**: Specific words or phrases
- **Phone Numbers**: Caller or recipient
- **Variables**: Variable values
- **Errors**: Error messages

## **Exporting Logs**

### **Export Options**

- **CSV Export**: Spreadsheet format
- **JSON Export**: Structured data
- **PDF Export**: Document format
- **API Access**: Programmatic access

## **Best Practices**

### **1. Regular Review**

Review logs regularly:
- Daily for new deployments
- Weekly for established agents
- After incidents
- For optimization

### **2. Monitor Errors**

Watch for errors:
- Failed actions
- Timeouts
- Connection issues
- Configuration problems

### **3. Track Performance**

Monitor metrics:
- Response times
- Success rates
- User satisfaction
- Error rates
